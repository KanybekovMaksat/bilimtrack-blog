# Технический анализ проекта Bilimtrack Blog

Этот документ содержит детальный анализ архитектуры, структуры, функционала, проблемных зон и необходимых доработок для проекта **Bilimtrack Blog**.

---

## 1. Описание проекта

**Bilimtrack Blog** — это контентный хаб, блог и лидогенератор для образовательной экосистемы **Bilimtrack** (SaaS-платформа для автоматизации учебных заведений: школ, колледжей и вузов в Кыргызстане и СНГ).

*   **Основная цель**: Привлечение руководителей учебных заведений через полезный контент (кейсы внедрения, советы по управлению, новости) и конвертация их в лиды (заявки на демо-версию системы).
*   **Локализация**: Контент ориентирован на СНГ (основной язык русский). В API заложена мультиязычность (`ru`, `ky`, `en`), но интерфейс на данный момент поддерживает только русский язык.

---

## 2. Технологический стек

Проект построен на современном и производительном веб-стеке:
*   **Фреймворк**: [Next.js 16](file:///f:/Programming/Big%20Projects/bilimtrack-blog/package.json) (используется классический **Pages Router**, а не App Router).
*   **Рендеринг**: React 19, TypeScript 5.6.
*   **Стилизация**: [Tailwind CSS v4](file:///f:/Programming/Big%20Projects/bilimtrack-blog/package.json) + библиотека компонентов **HeroUI v3** + `next-themes` (для работы с темами).
*   **Редактор (WYSIWYG)**: **BlockNote** (`@blocknote/*`) — используется в административной панели для написания и редактирования статей.
*   **Архитектурный паттерн**: **Feature-Sliced Design (FSD)**. Вся логика разделена по слоям (`shared`, `entities`, `features`, `widgets`, `pages`).

## 3. Обзор функционала сайта

Проект разделен на две основные части: публичную (для читателей) и административную (CMS для редакторов).

### Публичная часть (Блог)
1.  **Главная страница блога (`/blog`)**:
    *   Слайдер/Hero-секция с закрепленной (featured) статьей.
    *   Фильтрация статей по категориям («Кейсы», «Управление», «Советы», «Новости»).
    *   Поиск статей по ключевым словам (выполняется на стороне сервера через API).
    *   Лента статей с пагинацией (кнопка «Загрузить еще»).
    *   Боковая панель (сайдбар) с популярными статьями и кнопкой призыва к действию (CTA) «Получить демо».
2.  **Страница чтения статьи (`/blog/[slug]`)**:
    *   Индикатор прогресса чтения статьи вверху страницы.
    *   Динамическое оглавление (TOC), генерируемое на основе заголовков `<h2>` из текста статьи.
    *   Возможность поделиться статьей в соцсетях.
    *   **Лид-магнит**: Блок подписки на рассылку для скачивания «Чек-листа цифровизации школы» (отправляет email на бэкенд как демо-заявку).
    *   Форма в конце статьи для прямой отправки заявки на демо-версию.

### Административная часть / CMS (`/writer/*`)
1.  **Страница входа (`/writer/login`)**:
    *   Авторизация с использованием JWT-токенов (`access` и `refresh`), которые сохраняются в `localStorage`. Автоматическое обновление токена (refresh) при истечении срока действия (ошибка `401`).
2.  **Панель статей (`/writer/articles`)**:
    *   Табличный список всех статей (черновики, опубликованные, архивированные).
    *   Фильтрация по статусам статей и поиск по заголовкам.
    *   Удаление статей и быстрый переход к редактированию или просмотру на сайте.
3.  **Редактор статей (`/writer/admin`)**:
    *   Визуальный WYSIWYG редактор (BlockNote) с поддержкой форматирования текста, списков, цитат, изображений.
    *   Автосохранение черновика в `localStorage` (ключ `bt_draft`) каждые 1.2 секунды + фоновая отправка автосохранений на бэкенд через API (если пользователь авторизован).
    *   Правая панель настроек статьи: установка заголовка, Slug (генерируется автоматически, но редактируется вручную), выбор категории и автора, выбор обложки (готовые стили-пресеты или загрузка файла), дата публикации, SEO-теги (`meta title`, `meta description`).
4.  **Вспомогательные страницы CMS**:
    *   Управление авторами (`/writer/authors`), категориями (`/writer/categories`), медиа-файлами (`/writer/media`), настройками (`/writer/settings`) и статистика просмотров (`/writer/stats`).

## 4. Что сейчас не работает или работает некорректно (Баги и Проблемы)

В ходе анализа кода были выявлены ключевые недоработки и баги, которые необходимо исправить перед запуском в продакшн:

> [!WARNING]
> **1. Критическая проблема с SEO на главной странице блога (`/blog`)**
> В файле [pages/blog/index.tsx](file:///f:/Programming/Big%20Projects/bilimtrack-blog/pages/blog/index.tsx) метод `getStaticProps` возвращает пустой объект `props: {}`. Данные загружаются на клиенте через `useEffect` в хуке [useArticleFeed](file:///f:/Programming/Big%20Projects/bilimtrack-blog/src/features/filter-articles/model/useArticleFeed.ts).
> *   **Следствие**: Поисковые роботы (Google, Yandex) видят пустую страницу без статей, что делает SEO-продвижение блога неэффективным. Загрузка данных должна быть перенесена в `getStaticProps` на сервере.

> [!CAUTION]
> **2. Баг редиректа на главной странице (`/`)**
> В файле [pages/index.tsx](file:///f:/Programming/Big%20Projects/bilimtrack-blog/pages/index.tsx) реализована логика проверки авторизации: если токена админа нет, страница редиректит на `/writer/login`.
> *   **Следствие**: Любой обычный пользователь, перейдя по адресу `https://bilimtrack.com/` (корневой домен), вместо главной страницы блога попадает на форму логина для администратора. Роут `/` должен безусловно перенаправлять обычных пользователей на `/blog`.

> [!IMPORTANT]
> **3. Жёстко захардкоженные данные (МУИТ кейс-фоллбек)**
> *   В [ArticleBody.tsx](file:///f:/Programming/Big%20Projects/bilimtrack-blog/src/widgets/article-content/ui/ArticleBody.tsx) при отсутствии HTML-контента из API отрисовывается массивный захардкоженный текст про переход университета МУИТ на цифровое управление.
> *   В [ArticlePage.tsx](file:///f:/Programming/Big%20Projects/bilimtrack-blog/src/pages/article/ui/ArticlePage.tsx) список популярных статей жестко прописан как пустой массив `popular: Article[] = [];`, из-за чего правый сайдбар всегда пуст.

> [!NOTE]
> **4. Имитация (Mock) загрузки обложек в CMS**
> Метод `uploadCoverImage` в [blog-api.ts](file:///f:/Programming/Big%20Projects/bilimtrack-blog/src/shared/api/blog-api.ts#L268-L277) не отправляет файл на бэкенд. Вместо этого он имитирует задержку сети и возвращает временную локальную ссылку `URL.createObjectURL(file)`. Эта ссылка ломается сразу после перезагрузки страницы или на устройствах других пользователей. Нужен полноценный API-эндпоинт для загрузки медиа.
